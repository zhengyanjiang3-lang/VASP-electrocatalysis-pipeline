# VASP Calculation Workflow

This skill automates the preparation of all input files needed for VASP structure optimization and ZPE calculations on Cu surface adsorption systems.

## Root Directory File Conventions

The following files should already exist in the root directory (this skill does not create them):

- `INCAR-opt`: INCAR for structure optimization
- `INCAR-zpe`: INCAR for ZPE calculation
- `sub.sh`: Job submission script template

## Folder Creation Rules

- If the user says "I want to calculate structure optimization for structure B on metal A" or "如果我说我想算A金属的B文件的结构优化", create the folder `A/B` (i.e., create subfolder `B` under the metal `A` directory).
- If the user says "I want to calculate ZPE for structure B on metal A" or "我想算A金属的B结构的zpe", create the folder `A/B-zpe` (i.e., create subfolder `B-zpe` under the metal `A` directory).
- The metal directory (e.g., `Cu/`, `Ni/`) serves as the parent folder. All calculation folders for that metal are organized underneath it.


## 0. Structure Generation (before opt)

Before running opt, use a prompt-driven step to auto-build the initial POSCAR for each site.

**Prompt template:**

```
On the slab ({SLAB_POSCAR}) of metal {METAL}, place the adsorbate {ADSORBATE} ({SOURCE}).
Use pymatgen's AdsorbateSiteFinder to enumerate four sites: top / bridge / fcc / hcp.

Keep the slab as-is. Set the bottom two metal layers to F F F and all other atoms to T T T,
and include the Selective dynamics line.
All interatomic distances must be physically reasonable; no two atoms may be too close
(default minimum distance >= 0.8 x sum of covalent radii; metal-adsorbate atom >= 1.8 A).
If a site fails this check, automatically raise the adsorption height and retry.
```

Output: `{METAL}/{ADSORBATE}-{site}/POSCAR`. The generated structures feed directly into the opt workflow below.

## I. Structure Optimization Workflow (opt)

When the user requests structure optimization for one or more folders, follow these steps:

### 1. Prepare INCAR

```bash
cp INCAR-opt <target_folder>/INCAR
```

### 2. Prepare POSCAR

- The specific content and source of the POSCAR will be specified by the user at runtime.
- **Critical: The bottom two layers of Cu atoms must be fixed.**
  - Read the POSCAR and identify Cu atom coordinates.
  - Sort Cu atoms by z-coordinate; set the selective dynamics of the two layers with the smallest z-values to `F F F`.
  - Set all other atoms to `T T T`.
  - If the POSCAR does not have a selective dynamics line, add one.
- **How to determine layers:** Sort Cu atoms by z-coordinate. The two groups with the smallest z-values are the bottom two layers.

### 3. Generate KPOINTS and POTCAR

```bash
cd <target_folder>
echo -e "102\n1\n0.03" | vaspkit
```

> Note: Do not add an `n` prefix. The vaspkit interactive input takes plain numbers.

### 4. Verify POSCAR–POTCAR Consistency

- Read line 6 of POSCAR (element types) and line 7 (atom counts).
- Read all `TITEL` lines in POTCAR to extract the element order.
- **Confirm that the element types and their order match exactly.** If they do not match, report the discrepancy to the user.

### 5. Prepare sub.sh

- Copy the root directory `sub.sh` into the target folder.
- **Ask the user** for the required walltime.
- Set the job name to the format `Metal-FolderName`.
  - Example: folder `HCO`, metal `Cu` → job name `Cu-HCO`.
  - Identify the metal name from the POSCAR, or ask the user.

### 6. Checklist

After completion, verify the target folder contains:
- [ ] INCAR (from INCAR-opt)
- [ ] POSCAR (bottom two Cu layers fixed)
- [ ] KPOINTS (generated by vaspkit)
- [ ] POTCAR (generated by vaspkit, element order matches POSCAR)
- [ ] sub.sh (walltime and job name configured)

## II. ZPE Calculation Workflow

When the user requests ZPE calculations, the folder naming convention is `X-zpe`, where `X` is the corresponding opt folder name.

### 1. Prepare POSCAR

```bash
cp <X_folder>/CONTCAR <X-zpe_folder>/POSCAR
```

- Copy the CONTCAR from the completed opt calculation folder as the POSCAR.

### 2. Fix Atoms

- ZPE calculations require **fixing all atoms except the adsorbate molecule**.
- The adsorbate molecule consists of elements: **H, C, O** (i.e., HCO).
- Procedure:
  - Read the element types of all atoms in the POSCAR.
  - Set all atoms that are **not** H, C, or O to `F F F`.
  - Set all atoms that **are**  H, C, or O to `T T T`.
  - If the POSCAR does not have a selective dynamics line, insert `Selective dynamics` between the atom count line and the coordinate type line.

### 3. Prepare INCAR

```bash
cp INCAR-zpe <X-zpe_folder>/INCAR
```

### 4. Generate KPOINTS and POTCAR

```bash
cd <X-zpe_folder>
echo -e "102\n1\n0.03" | vaspkit
```

### 5. Verify POSCAR–POTCAR Consistency

- Same as the opt workflow: read the element line in POSCAR and the TITEL lines in POTCAR.
- **Confirm that the element types and their order match exactly.**

### 6. Prepare sub.sh

- Copy the root directory `sub.sh` into the target folder.
- **Ask the user** for the required walltime.
- Set the job name to the format `Metal-FolderName`.
  - Example: folder `HCOO-zpe`, metal `Cu` → job name `Cu-HCOO-zpe`.

### 7. Checklist

After completion, verify the target folder contains:
- [ ] INCAR (from INCAR-zpe)
- [ ] POSCAR (from the X folder's CONTCAR, only HCO atoms unfixed)
- [ ] KPOINTS (generated by vaspkit)
- [ ] POTCAR (generated by vaspkit, element order matches POSCAR)
- [ ] sub.sh (walltime and job name configured)

## General Notes

### Convergence Check Rule

When judging whether an optimization has converged before using its `CONTCAR` for downstream or ZPE calculations, check `vasp.log`, not `OUTCAR` alone. The final line or the second-to-last line of `vasp.log` must contain the convergence marker:

```text
reached required accuracy - stopping structural energy minimisation
```

If this marker is absent from the last two lines of `vasp.log`, do not treat the optimization as converged.

### POSCAR Selective Dynamics Format

```
System name
1.0
  ax ay az
  bx by bz
  cx cy cz
Element1 Element2 ...
Count1   Count2   ...
Selective dynamics        ← This line must be present
Direct (or Cartesian)
x1 y1 z1 T T T (or F F F)
x2 y2 z2 T T T (or F F F)
...
```

### Determining Which Element an Atom Belongs To

Line 6 of the POSCAR lists the element types, and line 7 lists the number of atoms for each element. Atom coordinates follow this order. For example:

```
Cu O C H
48  2 1 1
```

This means the first 48 atoms are Cu, the next 2 are O, then 1 C, and 1 H.

### Information to Confirm with the User at Runtime

1. **POSCAR source/content** (for opt calculations)
2. **Walltime** (for sub.sh)
3. **Metal name** (if it cannot be clearly determined from the POSCAR)
4. **POSCAR–POTCAR mismatch** — report and wait for user instructions

## III. Convergence Check and Energy Summary Workflow

When the user requests a convergence check and energy summary across metal folders, follow these steps:

### 1. Convergence Check

- Scan **all non-zpe folders** (i.e., folders whose names do not end with `-zpe`) under each metal directory.
- For each folder, check `vasp.log`: the **last line or the second-to-last line** must contain:

  ```text
  reached required accuracy - stopping structural energy minimisation
  ```

- If **all folders are converged**, proceed to step 2 (energy summary).
- If **any folders are not converged**:
  - Report a list of the unconverged folders.
  - Investigate and report the likely reason for non-convergence (e.g., reached `NSW` step limit, electronic SCF not converged, job killed, missing `vasp.log`, etc.).
  - **Wait for the user's next instruction.** Do not proceed to step 2 until the user explicitly says to proceed (e.g., "可以统计了" / "go ahead and summarize").

### 2. Identify Minimum-Energy Structure (after user approval)

- For each metal folder, among its converged non-zpe subfolders, identify the structure with the **lowest energy**.
- **Energy is taken from the last line of `OSZICAR`, the `E0` value.**
- Report: for each metal, which folder has the minimum energy and what that energy is.

### 3. Build the `data.xlsx` Summary Table

Create a new file `data.xlsx` in the project root with the following layout:

- **Header row (row 1):** `中心金属 | Energy | ZPE | TS | G | ΔG`
  - These header cells must be filled with **light blue** background.
- **Subsequent rows:** one row per converged non-zpe structure across all metals.
  - Column 1 (`中心金属`): the structure folder name (e.g., `HCO`, `NHO`).
  - Column 2 (`Energy`): the `E0` from `OSZICAR` (last line) of the opt folder.
  - Column 3 (`ZPE`): from the corresponding `<folder>-zpe` directory (see below).
  - Column 4 (`TS`): from the corresponding `<folder>-zpe` directory (see below).
  - Column 5 (`G`): computed as `G = Energy + ZPE - TS`.
  - Column 6 (`ΔG`): leave blank unless the user specifies a reference.

- If the table needs more rows or columns than initially allocated, add them as needed.
- Group structures by metal (e.g., all `Ni/*` rows together, then `Cu/*`, etc.), or follow any grouping the user specifies.

### 4. Extract ZPE and TS

For each `<folder>-zpe` directory, run inside that folder:

```bash
printf "501\n298.15\n" | vaspkit | grep -E "Zero-point energy E_ZPE|Entropy contribution T\*S"
```

Sample output:

```text
Zero-point energy E_ZPE   :      13.814 kcal/mol    0.599026 eV
Entropy contribution T*S  :   15147.343 J/(mol)     0.156991 eV
```

- **ZPE** = the value in the **second-to-last column** of the `Zero-point energy E_ZPE` line (in eV), e.g., `0.599026`.
- **TS** = the value in the **second-to-last column** of the `Entropy contribution T*S` line (in eV), e.g., `0.156991`.

### 5. Compute G

For each row:

```text
G = Energy + ZPE - TS
```

Fill the result into the `G` column.

### 6. Formatting Notes

- The header cells (`金属`, `Energy`, `ZPE`, `TS`, `G`, `ΔG`) must be styled with a **light blue** background fill.
- Use `openpyxl` (or equivalent) to create the `.xlsx` file with the cell fill applied.
